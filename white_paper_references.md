## White Paper References

The following sources provide background and supporting evidence for the concepts introduced in the Cognexis white paper.  Each reference includes a citation to an external source using the format prescribed for citations.

1. **Deep Equilibrium Models (DEQ).**  This chapter on deep equilibrium models introduces implicit layer models and explains how a single layer can be reused until a fixed point is reached.  It notes that DEQ models have demonstrated impressive performance on a variety of large‑scale vision and NLP tasks, and that the core idea originates from early work on recurrent backpropagation【106536656981761†L10-L22】.  The tutorial further describes how DEQ models extend traditional architectures by expressing the entire network as an equilibrium computation, and clarifies that weight tying and input injection are key design elements to ensure convergence and input dependence【106536656981761†L49-L59】.

2. **Quadratic Complexity of Transformers.**  In this article, Pedram Hosseini explains why the vanilla self‑attention mechanism in transformers has quadratic complexity in sequence length.  Because self‑attention computes pairwise interactions between all tokens, the dot‑product operation, softmax normalization and value aggregation each require \(O(n^2)\) computation and memory, where \(n\) is the sequence length.  Consequently, both compute and memory scale quadratically with sequence length【819387035014610†L37-L50】.  The article also explains that doubling the input length makes computation four times slower and that variants such as sparse and low‑rank attention aim to mitigate this limitation【819387035014610†L54-L70】.

3. **Adaptive Computation Time (ACT).**  Alex Graves’ 2016 paper introduces the Adaptive Computation Time algorithm for recurrent neural networks.  ACT allows a recurrent network to learn how many computational steps to take on each input and output.  The abstract states that ACT is deterministic, differentiable and requires minimal changes to the architecture.  It adapts the number of steps to the difficulty of the problem, allocating more computation to harder‑to‑predict transitions while maintaining efficiency【821717946454232†L51-L64】.  This mechanism inspires the compute‑adaptive loop scheduler in Cognexis.
